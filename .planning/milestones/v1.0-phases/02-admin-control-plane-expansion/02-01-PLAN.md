- Control-plane state is persisted in `system_configs`, but create-time enforcement happens before room objects are inserted into in-memory managers.

</trust_boundaries>

<stride_register>

  category: Spoofing
  component: admin capability/runtime endpoints
  disposition: mitigate
  mitigation_plan: Require admin auth on `GET/PATCH /api/admin/capabilities` and `GET/PATCH /api/admin/runtime`, and cover the auth boundary in `test-logic/admin-control-plane.test.js`.
- threat_id: T-01-02
  category: Tampering
  component: persisted control-plane state
  disposition: mitigate
  mitigation_plan: Allowlist game keys and runtime keys in `lib/admin/control-plane.js`, reject unknown fields and invalid value types, and record normalized before/after snapshots.
- threat_id: T-01-03
  category: Denial of Service
  component: room creation boundary
  disposition: mitigate
  mitigation_plan: Enforce capability and maintenance checks only in create handlers so future rooms are blocked intentionally while existing rooms keep their in-memory state untouched.
</stride_register>
</threat_model>
