<threat_model>
<trust_boundaries>
- Universal room entry crosses from a public room-number/share-link surface into family-specific join handlers and live room managers.
- Guest tokens travel through both HTTP handlers and Socket.IO handshake state; over-broad guest claims would become an authorization escape hatch.
- Post-match guest claim data crosses from ephemeral room/session state into persistent database rows.
</trust_boundaries>
