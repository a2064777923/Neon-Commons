<threat_model>
<trust_boundaries>
- Admin runtime controls directly affect live player behavior, so an imprecise scope model could block more rooms than intended.
- Hub, entry, and party-room pages all depend on the same degraded-state contract; selector drift or copy drift would make the operator story and player story diverge.
- Voice is a mixed backend/browser concern; the backend must define the status semantics while the browser app adds local device detail on top.
</trust_boundaries>
