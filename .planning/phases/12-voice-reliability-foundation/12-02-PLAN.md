<threat_model>
<trust_boundaries>
- Client-generated degradation reports influence room-wide transport mode, so the server must validate and serialize the authoritative result.
- Voice reconnect behavior crosses both socket lifecycle and page lifecycle; sloppy state handling can create ghost peers, duplicate seats, or accidental hot-mic recovery.
- Undercover is a shared transport consumer with stricter speaking rules than general party rooms, so shared runtime code must not erase page-specific permission checks.
</trust_boundaries>

<stride_register>
- threat_id: T-12-04
  category: Availability
  component: relay fallback switching
  disposition: mitigate
  mitigation_plan: switch once into `relay-required` on startup timeout or persistent degradation and keep relay sticky for the room visit to avoid oscillation loops.
- threat_id: T-12-05
  category: Spoofing
  component: reconnect recovery speaking state
  disposition: mitigate
  mitigation_plan: always recover with `muted: true` and leave unmute authority to explicit user action plus Undercover turn gating.
- threat_id: T-12-06
  category: Availability
  component: peer lifecycle cleanup
  disposition: mitigate
  mitigation_plan: centralize peer teardown, reconnect timers, and relay rebuild logic in one shared runtime and one backend manager instead of duplicate page-specific implementations.
</stride_register>
</threat_model>
