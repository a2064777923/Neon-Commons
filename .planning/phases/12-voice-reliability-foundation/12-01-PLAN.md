# Phase 12 Plan 01: Introduce relay/fallback voice transport contracts and configuration

<objective>
Create the backend-owned contract layer that Phase 12 execution will depend on.

Purpose: the repo currently has WebRTC voice signaling, reconnecting room seats, and truthful degraded voice messaging, but it still lacks one typed contract for relay fallback, reconnect defaults, and room-visit transport state. This plan introduces that contract through the existing config, shared-network, availability, and party-room serialization paths.
Output: typed voice transport config, additive relay/recovery enums and payload fields, and contract tests that lock the room-detail surface down before runtime behavior changes land.
</objective>

<threat_model>
<trust_boundaries>
- Voice transport policy crosses the backend/browser boundary; if the page owns the truth, reconnect and multi-user relay behavior will drift.
- The room-detail payload is public to seated players and guests, so transport contract fields must stay additive and must not leak broader admin-only config internals.
- Existing Phase 11 degraded-mode controls already influence player behavior; relay fallback must compose with them instead of silently overriding them.
</trust_boundaries>

<stride_register>

  category: Tampering
  component: relay/fallback transport policy
  disposition: mitigate
  mitigation_plan: centralize transport enums, report reasons, and normalized defaults in shared config and network-contract helpers instead of page-local constants.
- threat_id: T-12-02
  category: Availability
  component: typed relay configuration
  disposition: mitigate
  mitigation_plan: normalize `partyVoiceTransport` through `lib/system-config.js` so missing or malformed thresholds fall back to safe defaults rather than disabling voice recovery.

  category: Information Disclosure
  component: room voice transport payload
  disposition: mitigate
  mitigation_plan: serialize only browser-required ICE-server fields and public relay/recovery mode data; do not expose persistence metadata or raw diagnostics blobs.
