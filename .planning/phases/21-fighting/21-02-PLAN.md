        - Set attacker.hitThisAttack = true

    **lib/fighting/physics.js** — Platform physics per D-14, D-15, D-16 from 21-CONTEXT.md:

    1. Export applyGravity(character, dt):
       - `character.velocity.y += GRAVITY * dt`
       - `character.pos.y += character.velocity.y * dt`

    2. Export resolvePlatformCollision(character, arena):
       - Only check when character.velocity.y > 0 (falling, not rising)
       - For each platform in arena.platforms:
         - Calculate character bottom: `charBottom = character.pos.y + DEFAULT_HURTBOX.y + DEFAULT_HURTBOX.h`
         - Calculate character left/right: `charLeft = character.pos.x + DEFAULT_HURTBOX.x`, `charRight = charLeft + DEFAULT_HURTBOX.w`
         - Check horizontal overlap: `charRight > plat.x && charLeft < plat.x + plat.w`
         - Check vertical: `charBottom >= plat.y && charBottom <= plat.y + 15` (15px tolerance)
         - On collision: snap character to platform top, set velocity.y = 0, set grounded = true
         - If character was in "fall" state: transition to "idle"

    3. Export checkRingOut(character, arena):
       - Return `character.pos.y > arena.bounds.bottom + 100`
       - The +100 buffer prevents false positives near platform edges (per Pitfall 6)

    **lib/fighting/delta.js** — Delta-state computation for 60Hz (adapted from lib/racing/delta.js pattern):

    4. Export computeDelta(room, previousState):
       - Iterate room.characters
       - For each character, compute serialized state:
         ```javascript
         {
           seatIndex, state, frameCount,
           pos: { x: round1(pos.x), y: round1(pos.y) },
           velocity: { x: round1(vel.x), y: round1(vel.y) },
           facing, health, energy,
           grounded, invulnerable
         }
         ```
       - Compare with previousState (if exists):
         - Calculate position diff: `Math.abs(pos.x - prev.pos.x) + Math.abs(pos.y - prev.pos.y)`
         - If posDiff < POSITION_THRESHOLD (2.0 pixels for 60Hz) AND state unchanged AND health unchanged: skip
         - Always include if state changed (attack, hit_stun, ko, ring_out)
       - Return:
         ```javascript
         {
           tick: room.tick,
           fightPhase: room.fightPhase,
           countdown: room.countdownValue || 0,
           characters: includedCharacters,
           roundWins: [...]
         }
         ```
       - Use round1(n) helper: `Math.round(n * 10) / 10` (1 decimal for 60Hz, less precision than Racing's 3 decimals — fighting positions are pixel-based, not physics-based)

    5. Add POSITION_THRESHOLD = 2.0 (higher than Racing's 0.01 because 60Hz means smaller per-tick deltas; per Pitfall 3)
