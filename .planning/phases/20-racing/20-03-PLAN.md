         - Keyboard: useEffect with keydown/keyup listeners for arrow keys
         - Track key state: { up: false, down: false, left: false, right: false }

         - Send via socket: socket.emit("racing:input", { roomNo, input })
         - 20Hz throttle: only send if 50ms since last send
       - Ready button: socket.emit("racing:ready", { roomNo, ready: true })
       - Render: gameShell div containing RacingScene, HUD, TouchControls
       - Pass gameState, mySeatIndex, input state to children

    **components/racing/RacingScene.js:**

    3. Create Three.js scene component:
       - Import THREE (three)
       - On mount (useEffect with empty deps):
         - Create WebGLRenderer: `new THREE.WebGLRenderer({ antialias: true, canvas: canvasRef.current })`
         - Set size to window dimensions, pixelRatio to window.devicePixelRatio
         - Create Scene with background color #0a0e1a
         - Create PerspectiveCamera (fov 75, near 0.1, far 1000)
         - Create lights: DirectionalLight (white, intensity 1.2, position above), AmbientLight (soft, intensity 0.4)
         - Add TrackModel to scene
         - Add CarModel for each player (up to 4)
         - Start animation loop (requestAnimationFrame)
         - Handle window resize
       - On gameState update (useEffect):
         - Update car positions and rotations from gameState.cars
         - Use THREE.Vector3.lerp for position interpolation
         - Use THREE.Quaternion.slerp for rotation interpolation
         - Update camera to follow player's car (third-person chase):
           - Camera position = car position + offset behind and above
           - Camera looks at car position
           - Smooth follow using lerp (factor 0.1)
         - If phase is "countdown": show countdown text overlay
         - If phase is "finished": show result overlay
       - Cleanup: dispose renderer, geometries, materials on unmount

    **components/racing/TouchControls.js:**

    3. Create mobile touch controls:
       - Detect touch device: `const isTouchDevice = typeof window !== 'undefined' && ('ontouchstart' in window || navigator.maxTouchPoints > 0)`
       - If not touch device, render nothing (return null)
       - Left side: Virtual joystick (div with touch event handlers)
         - onTouchStart: record start position
         - onTouchMove: compute delta from start, map to steer value (-1..1)
         - onTouchEnd: reset steer to 0
         - Visual: circle within circle, inner moves with touch
       - Right side: Two buttons (accel and brake)
         - Accel button: onTouchStart sets accel=1, onTouchEnd sets accel=0
         - Brake button: onTouchStart sets brake=1, onTouchEnd sets brake=0
         - Visual: circular buttons with arrow icons
       - Call onInput callback with { accel, brake, steer } on any change

    **Update pages/racing/[roomNo].js:**

    4. Add client-side prediction:
       - Track local input state and last sent input
       - On each animation frame (in RacingScene): if input changed, apply locally using simplified physics (forward movement based on accel, rotation based on steer)
       - Store predicted position separately from server position
       - On receiving server state (racing:update):
         - For own car: if server position differs from predicted by > threshold (2 units), snap to server position. Otherwise, lerp toward server position (reconciliation)
         - For other cars: snap to server position immediately (no prediction needed)

    5. Add interpolation for smooth rendering:
       - Store last two server states for each car
       - On each render frame: interpolate between last two states based on elapsed time since last update
       - Use THREE.Vector3.lerp for position, THREE.Quaternion.slerp for rotation
       - Interpolation factor = (now - lastUpdateTime) / 50ms (one tick period)

    6. Wire HUD:

    7. Wire TouchControls:

       - Same 20Hz throttle as keyboard input

    8. Add race result overlay:
       - When racePhase === "finished": show overlay with winner name and "Back to Hub" button
       - Button navigates to "/"
