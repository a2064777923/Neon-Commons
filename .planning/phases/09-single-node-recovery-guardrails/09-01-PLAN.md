<threat_model>
<trust_boundaries>
- The shared room directory is trusted by room-entry resolve/shareable/guest handlers, so persisted snapshot corruption would affect public discovery.
- Backend startup crosses a trust boundary from database rows into live runtime memory; invalid rows must not become fake live rooms.
- Private room membership lists are persisted for shareable discovery, so the snapshot schema must not expose more identity detail than the current directory already owns.
</trust_boundaries>
